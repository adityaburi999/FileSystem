/objects/

            91b2ff10.chunk

========================================================
/cache/
========================================================

/cache/
    /ram/
    /ssd/
    /reconstructed/

→ Temporary performance layer
→ Stores hot chunks and reconstructed files
→ Fully disposable (can be wiped anytime)

========================================================
/staging/
========================================================

/staging/
    txn_001/
        buffer.tmp
        chunks.tmp
        redirect.tmp

→ Temporary write area
→ Holds incomplete writes
→ NOT visible to filesystem
→ Deleted on crash or commit

========================================================
/system/
========================================================

/system/
    /gc/
    /snapshots/
    /index/
    /state/

→ Internal engine data
→ WAL checkpoints
→ GC state tracking
→ Indexing structures

→ Hidden from user filesystem (FUSE restriction)

========================================================
KEY DESIGN RULES
========================================================

1. WAL = append-only truth log
2. metadata = versioned filesystem state
3. objects = immutable data chunks
4. cache = disposable acceleration layer
5. staging = crash-safe temporary workspace
6. system = internal engine (not user accessible)
